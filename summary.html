IBN – Weekly Overview (Weeks 1–10)

Week 1 – Introduction to Operating Systems

  • What is an operating system? (resource manager, abstraction layer)
  • Interrupts (hardware/software)
  • User mode vs kernel mode
  • System calls (read, write, etc.)
  • Von Neumann architecture
  • Process vs program

Week 2 – Processes and Threads

  • Processes vs threads (isolation vs sharing)
  • Context switch (overhead and components)
  • Process states (ready, running, blocked)
  • Concurrency and non-determinism
  • Amdahl’s Law (limits of parallel speedup)

Week 3 – Thread Basics and Scheduling

  • User-level vs kernel-level threads
  • Thread scheduling strategies (round-robin, preemption)
  • Thread safety and race conditions
  • Critical sections and mutual exclusion

Week 4 – POSIX Threads and Race Conditions

  • Creating and joining threads (pthread_create, pthread_join)
  • Race conditions with shared data
  • Mutexes (pthread_mutex_t)
  • Simple deadlock scenarios
  • Thread lifecycle (join vs detach)

Week 5 – Synchronization and Semaphores

  • Producer-consumer problem (bounded buffer)
  • Semaphores: wait(), signal()
  • Counting vs binary semaphores
  • Implementing mutual exclusion and signaling

Week 6 – Condition Variables

  • pthread_cond_wait, pthread_cond_signal
  • Condition-based waiting with mutexes
  • Proper loop usage to avoid spurious wakeups
  • Comparison with semaphores

Week 7 – Higher-Level Coordination & Deadlocks

  • Barriers (pthread_barrier_t)
  • Reader-writer problem
  • Reader-writer locks (pthread_rwlock_t)
  • Coffman conditions for deadlock

Week 8 – Deadlock Detection & Banker’s Algorithm (Intro)

  • Resource Allocation Graphs (RAGs)
  • Belegungsmatrix (C), Anforderungsmatrix (R)
  • Safe vs unsafe system states
  • Banker’s Algorithm idea

Week 9 – Banker’s Algorithm and Simulation

  • Full simulation procedure (adjust A, C, R)
  • Detection vs avoidance
  • Execution path diagrams (A-horizontal, B-vertical)
  • Forbidden zones and deadlock regions

Week 10 – Scheduling Basics

  • What is scheduling? (dispatcher, strategy)
  • Types: batch, interactive, real-time
  • CPU-bound vs I/O-bound behavior
  • First-Come-First-Served (FCFS) and convoy effect

IBN – Week 1 Summary

Topics Covered:

  1. Betriebssysteme (Operating Systems):

    • Definition and functions:

      • Resource management
      • Abstract machine interface

    • Role of OS as “interpreter between hardware and software”

  2. Interrupts:

    • Used to signal and coordinate events (e.g. hardware input)

    • Types of interrupts:

      • Hardware interrupts (external)
      • Software interrupts (e.g. system calls)

  3. Modi (Modes):

    • User Mode vs Kernel Mode

      • User mode = restricted, cannot access hardware directly
      • Kernel mode = full privileges

  4. Systemaufrufe (System Calls):

    • Mechanism for user programs to request OS services
    • Example: read, write, fork, exec

  5. Von-Neumann Architektur:

    • Single memory for instructions and data
    • Stored-program concept

  6. Grundbegriffe:

    • Prozess: instance of a program in execution
    • Programm: static code, not executing
    • Other terms like Thread, CPU-Burst, and I/O-Burst introduced

Notes:

  • The week establishes a foundational understanding of what an operating system is and how it interfaces with hardware and programs.
  • Concepts like interrupts and modes introduce the control mechanisms the OS uses to manage execution and safety.
  • The use of system calls introduces how user programs interact with the kernel in a controlled way.

IBN – Week 2 Summary

Topics Covered:

  1. Prozesse und Threads (Processes and Threads):

    • Prozess:

      • Contains its own address space
      • Heavyweight unit of execution

    • Thread:

      • Shares address space with other threads of the same process
      • Lightweight and more efficient in context switches

  2. Kontextwechsel (Context Switch):

    • Saving and restoring state when switching between processes or threads
    • Overhead: higher for processes (full memory, registers) vs. threads (only registers, stack)

  3. Prozesszustände (Process States):

    • Running, Ready, Blocked (Waiting), Terminated
    • Introduced the state transition diagram

  4. Modell der Parallelität:

    • Concurrent vs. parallel execution
    • Interleaving and synchronization issues
    • Non-determinism in thread scheduling

  5. Amdahl’s Law (Kurz):

    • Limits of parallelism due to serial portions of a program

Notes:

  • Week 2 transitions from OS abstraction to execution units and how they are managed.
  • Highlights the differences between threads and processes, and introduces essential synchronization and performance topics.

IBN – Week 3 Summary

Topics Covered:

  1. Thread-Konzepte (Thread Concepts):

    • User-level threads vs Kernel-level threads:

      • User-level: managed by the application, more lightweight, but kernel unaware
      • Kernel-level: managed by OS, allows true parallelism on multi-core systems

  2. Thread Scheduling & Fairness:

    • Round-Robin and time-slicing introduced
    • Preemptive vs non-preemptive scheduling
    • Concept of “fairness” in distributing CPU time across threads

  3. Thread Safety:

    • Race conditions when threads access shared resources unsynchronized
    • Determinism vs nondeterminism in thread execution

  4. Kritische Abschnitte (Critical Sections):

    • Where shared state is accessed/modified
    • Requires protection to avoid inconsistencies

  5. Mutual Exclusion:

    • Prevents simultaneous access to critical sections
    • Implemented via locks, mutexes, or atomic operations

Notes:

  • This week deepens the discussion on multithreading, emphasizing issues of concurrency, correctness, and scheduling behavior.
  • Sets the foundation for synchronization mechanisms, which are crucial for upcoming weeks on semaphores, condition variables, etc.

IBN – Week 4 Summary

Topics Covered:

  1. POSIX Threads (pthreads):

    • Creation and joining of threads
    • Example of pthread_create() and pthread_join()
    • Passing arguments to threads
    • Joining threads ensures proper synchronization before exit

  2. Race Conditions Illustrated:

    • Classic example: incrementing a shared variable without synchronization
    • Use of sleep() to simulate interleaving effects
    • Demonstrates nondeterminism and the need for synchronization

  3. Mutual Exclusion with Mutexes:

    • Use of pthread_mutex_t
    • Locking (pthread_mutex_lock) and unlocking (pthread_mutex_unlock)
    • Protects critical sections to prevent race conditions

  4. Deadlock Example:

    • Simple case where two threads hold one mutex and wait for the other
    • Illustrates circular wait condition

  5. Thread Lifecycle Management:

    • Initial, running, terminated (joined or detached)
    • Importance of pthread_join() vs pthread_detach()

Notes:

  • Week 4 brings theory into practice with real code examples of multithreading in C using pthreads.
  • It provides hands-on understanding of race conditions, mutexes, and basic deadlock scenarios.

IBN – Week 5 Summary

Topics Covered:

  1. Synchronisationsprobleme (Classic Synchronization Problems):

    • Producer-Consumer Problem (Bounded Buffer):

      • Producer writes to buffer, consumer reads
      • Needs synchronization to avoid buffer overflow/underflow

    • Requires:

      • Mutual exclusion
      • Signaling when buffer is not full / not empty

  2. Semaphoren (Semaphores):

    • Two operations: wait() (P) and signal() (V)
    • Counting semaphores manage access to shared resources
    • Binary semaphores = mutexes

  3. Lösungen mit Semaphoren:

    • Example code using semaphores for producer-consumer
    • Use of empty, full, and mutex semaphores to coordinate actions

  4. Condition Variables (Kurz erwähnt):

    • Briefly introduced as another synchronization primitive
    • More flexible than semaphores in some cases

  5. Vermeidung von Busy Waiting:

    • Semaphores and condition variables avoid wasting CPU cycles

Notes:

  • Week 5 introduces semantics and implementation of classic synchronization using semaphores.
  • It shows how synchronization problems are solved in practice with clean and efficient primitives, moving beyond busy-waiting loops.

IBN – Week 6 Summary

Topics Covered:

  1. Condition Variables (Full Introduction):

    • Used with a mutex to wait for a condition to become true

    • Functions:

      • pthread_cond_wait(&cond, &mutex)
      • pthread_cond_signal(&cond) / broadcast

    • Thread releases mutex while waiting and reacquires it before returning

  2. Producer-Consumer with Condition Variables:

    • Revisited the bounded buffer problem
    • Condition variables used to signal when buffer is not full or not empty
    • Clean separation of synchronization logic from state transitions

  3. Best Practices:

    • Always use a loop with pthread_cond_wait to recheck the condition
    • Avoid spurious wakeups

  4. Comparing Semaphores vs Condition Variables:

    • Semaphores are more general but less expressive for specific conditions
    • Condition variables are safer and more structured for waiting on shared state

Notes:

  • Week 6 focuses on a cleaner, higher-level synchronization approach than semaphores.
  • Emphasizes robust, correct usage patterns for pthread_cond_* functions, which are critical in real multithreaded applications.

IBN – Week 7 Summary

Topics Covered:

  1. Thread-Koordination auf höherer Ebene:

    • Barriers:

      • All threads must reach the barrier before any can proceed
      • Useful for phase-based parallel computation

    • POSIX: pthread_barrier_t

  2. Reader-Writer Problem:

    • Multiple readers can access shared data simultaneously
    • Writers need exclusive access
    • Goal: Avoid starvation and maximize concurrency

  3. Reader-Writer Locks:

    • Specialized locks that distinguish between readers and writers
    • POSIX: pthread_rwlock_t

  4. Deadlocks Revisited:

    • Conditions for deadlock (Coffman conditions):

      1. Mutual Exclusion
      2. Hold and Wait
      3. No Preemption
      4. Circular Wait

    • Discussion of prevention/avoidance/detection strategies

Notes:

  • Week 7 introduces synchronization patterns for more advanced coordination needs.
  • Focuses on fairness and efficiency when multiple threads interact with shared resources under different access modes.

IBN – Week 8 Summary

Topics Covered:

  1. Deadlock Detection and Resource Allocation Graphs (RAGs):

    • Processes = circles, resources = squares

    • Edges from process to resource = request

    • Edges from resource to process = allocation

    • Cycle in graph → potential deadlock

      • Single instance per resource: cycle ⇒ deadlock
      • Multiple instances: cycle ⇒ deadlock possible, not guaranteed

  2. Belegungsmatrix und Anforderungsmatrix:

    • C (Current allocation): how many resources each process holds
    • R (Request/need): how many more each process will request
    • E (Existing) and A (Available) resource vectors

  3. Banker’s Algorithm – Introduction:

    • Core idea: only grant a request if it keeps the system in a safe state
    • Simulate granting the request, check if all processes can still finish
    • If yes → approve; if no → deny or delay

  4. Safe vs Unsafe States:

    • Safe: some execution order allows all processes to finish
    • Unsafe: potential for deadlock exists (not necessarily immediate)

Notes:

  • Week 8 formally introduces graph-based modeling and avoidance strategies for deadlocks.
  • Emphasizes the predictive role of the Banker’s Algorithm and the importance of system state tracking.

IBN – Week 9 Summary

Topics Covered:

  1. Banker’s Algorithm (continued):

    • Detection vs Avoidance clarified:

      • Detection: analyze current state for existing deadlock
      • Avoidance: prevent entering unsafe state via simulation

  2. Simulation Steps (Recap):

    • Subtract request Reqᵢ from available vector A
    • Add Reqᵢ to allocation Cᵢ
    • Subtract Reqᵢ from remaining need Rᵢ
    • Then run Banker’s Algorithm to verify if system remains safe

  3. Graphical Visualization of Execution Paths:

    • 2D diagrams with process A (horizontal) and B (vertical)
    • Colored bands show when each process uses a resource
    • Forbidden zones: where mutual exclusion is violated
    • Deadlock zone: reachable, legal state where progress halts

  4. Deadlock Avoidance via Simulation:

    • Simulate request grant
    • If Banker’s Algorithm marks all processes: safe ⇒ approve
    • Else: block or delay the request

Notes:

  • Week 9 reinforces how dynamic prevention works via simulation before allocating resources.
  • Also introduces graphical resource trace analysis, a helpful way to understand deadlock zones visually and conceptually.

IBN – Week 10 Summary

Topics Covered:

  1. Scheduling Introduction:

    • Transition from resource management to CPU time allocation
    • Scheduling = deciding which process runs when on the CPU

  2. Types of Scheduling Environments:

    • Batch systems (e.g. scientific computing, no user interaction)
    • Interactive systems (e.g. desktops, laptops)
    • Real-time systems (e.g. embedded control, media)

  3. Scheduling Criteria:

    • Fairness
    • Response time (interactive)
    • Throughput
    • Deadline adherence (real-time)
    • Policy enforcement

  4. Process Behavior:

    • Alternates between CPU burst and I/O burst

    • Processes classified as:

      • CPU-bound: long CPU phases
      • I/O-bound: short CPU phases, frequent I/O

  5. First-Come-First-Served (FCFS):

    • Simple queue-based scheduling
    • Problem: convoy effect (long processes block short ones)
    • Leads to poor average waiting times

Notes:

  • Week 10 opens the Scheduling (A) unit by defining the motivation and environment for different scheduling policies.
  • Highlights how process behavior informs the choice of scheduling strategy.
